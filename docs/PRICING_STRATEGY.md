## 💡 PRICING PHILOSOPHY

### Core Principles

**1. Transparent (No Hidden Fees)**
```
✅ "€49/kuu, period"
❌ "€49/kuu + €10 per user + €20 per feature + ..."

Põhjendus:
- Trust building
- Easy decision
- No surprises
```

**2. Simple (3 Tiers Max)**
```
✅ FREE, STARTER, PROFESSIONAL
❌ 7 different tiers

Põhjendus:
- Analysis paralysis (liiga palju valikuid)
- Easier to explain
- Clearer upgrade path
```

**3. Fair (Pay for Value)**
```
✅ Flat fee (sõltumata organiast)
❌ Per-employee pricing (discourages growth)

Põhjendus:
- Predictable cost
- Growth-friendly
- Aligns with value (compliance, not per-head)
```

**4. Flexible (Monthly or Annual)**
```
✅ Cancel anytime (monthly)
✅ Save 17% (annual)

Põhjendus:
- Lower barrier (monthly)
- Better cash flow (annual)
- Customer choice
```

---

### Tactic 2: Charm Pricing

**€49 vs €50:**

```
€49/kuu feels significantly cheaper than €50/kuu
(even though difference is €1)

Why?
- Left-digit effect (4 vs 5)
- Perceived "under €50" (threshold)
- "Only €40-something"

Research: 20-40% higher conversion!
```

**Testing:**

```
A: €45/kuu (rounder, but less)
B: €49/kuu (charm price)
C: €50/kuu (round, psychological barrier)

Hypothesis: B (€49) wins
- Not too cheap (trust issue)
- Charm price effect
- Under €50 barrier
```

---

**3-Tier Pricing:**


## 🧪 PRICING EXPERIMENTS

### A/B Tests (Month 1-3)

**Test 1: Price Point**

```
Control: €49/kuu
Variant A: €39/kuu (-20%)
Variant B: €59/kuu (+20%)

Hypothesis: €49 is sweet spot

Measure:
- Conversion rate (FREE → STARTER)
- Revenue per visitor
- Customer lifetime value

Run for: 2 weeks (minimum 100 visitors per variant)
```

**Test 2: Annual Discount**

```
Control: 17% discount (€490/aasta)
Variant A: 20% discount (€470/aasta)
Variant B: 25% discount (€441/aasta)

Hypothesis: 20% drives more annual signups

Measure:
- Annual plan adoption rate
- Upfront cash
- Customer retention

Run for: 4 weeks
```

**Test 3: Trial Length**

```
Control: 30 days
Variant A: 14 days
Variant B: 60 days

Hypothesis: 30 days is optimal

Measure:
- Trial → Paid conversion
- Time to activation
- Product usage

Run for: 4 weeks
```

**Test 4: Pricing Page Layout**

```
Control: 3 columns (FREE, STARTER, PRO)
Variant A: 2 columns (STARTER, PRO - remove FREE)
Variant B: 4 columns (add ENTERPRISE)

Hypothesis: 3 columns (control) wins

Measure:
- Click-through rate
- Signup flow completion
- Mix (which tier chosen)

Run for: 2 weeks
```

## 💸 DISCOUNT STRATEGY

### When to Discount (Strategic)

**1. Launch Discount:**

```
Who: First 100 customers
What: 50% off first 3 months
Why: Generate buzz, early adopters

Example:
"🚀 Launch Special: €24.50/kuu (3 months)"
After 3 months: €49/kuu (grandfather clause)
```

**2. Annual Plan Discount:**

```
Who: Everyone
What: 17% off (2 months free)
Why: Cash flow, lower churn

€49/kuu × 12 = €588
Annual: €490 (save €98)
```

**3. Student/Non-Profit Discount:**

```
Who: Students, non-profits (verified)
What: 30% off
Why: Social good, market expansion

€49/kuu → €34/kuu
```

**4. Partner Discount:**

```
Who: IT consultants, resellers
What: 20% off (for their clients)
Why: Channel development

€49/kuu → €39/kuu (still profitable)
```

---

### When NOT to Discount

**❌ Sales objection:**
```
Customer: "Liiga kallis"
❌ BAD: "Ok, €39/kuu?"
✅ GOOD: "Saan aru. Mis on teie eelarve?
          Vaatame, kas saame sobitada."

Reason: Don't train customers to negotiate
```

**❌ Churn prevention:**
```
Customer: "Tahan tühistada"
❌ BAD: "Wait! 30% off!"
✅ GOOD: "Mis oli põhjus? Kuidas saaksime aidata?"

Reason: Fix problem, not price
```

**❌ Constant promotions:**
```
❌ BAD: "This month 20% off!"
       "Next month 30% off!"

Reason: Trains customers to wait for discount
```

---

### Coupon Codes (Selective)

**Use Cases:**

```
PARTNER20 → 20% off (partner referrals)
LAUNCH50 → 50% off 3 months (launch)
STUDENT30 → 30% off (students)
FRIENDS → 1 month free (referrals)
```

**Implementation:**

```
Stripe: Create coupon codes
Landing page: "Have a coupon code?" (small link)
Checkout: Enter code → discount applied

Track usage:
- Redemption rate
- Revenue impact
- Customer quality (do they churn more?)
```
